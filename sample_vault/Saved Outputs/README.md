# Saved Outputs

This folder is the primary home for app-generated outputs in the current sample-vault setup.

Suggested subfolders:
- `answers/General Asks/`
- `answers/Arrangement Plans/`
- `answers/Sound Design Brainstorms/`
- `research/`
- `critiques/Genre Fit Reviews/`
- `critiques/Track Concept Critiques/`
- `track_contexts/`

Use this for generated answers, critiques, and research runs you want to keep easy to browse without mixing them into curated knowledge or active project notes.

Saved answers can now also carry reviewable Track Context update proposals. When present, look for:
- frontmatter fields like `track_id`, `active_section_focus`, and `track_context_update`
- a `## Suggested Track Context Update` section in the note body

The intent is:
- answers stay easy to browse as outputs
- Track Context updates stay reviewable before persistence
- active in-progress track memory remains separate in `track_contexts/`
